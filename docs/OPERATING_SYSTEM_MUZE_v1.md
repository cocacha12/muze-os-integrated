# Operating System Muze v1

## Objetivo
Operar Muze con disciplina de empresa real: foco, deadlines, accountability y trazabilidad total.

## Canales
1. **DM Mark ↔ Gerencia IA (Kether):** estrategia, reglas, excepciones.
2. **Grupo Muze AI Consulting:** definición/validación de requerimientos y prioridades.
3. **Grupo Roadmap/Operación:** seguimiento, alertas, bloqueos, cumplimiento por responsable.

## Cadencia
- **Lunes:** Plan semanal (prioridades + capacidad).
- **Miércoles:** Control de bloqueos.
- **Viernes:** Cierre semanal (hechos, no hechos, desvíos, siguientes pasos).

## Reglas de tareas (DoD mínimo)
Toda tarea debe tener:
- Responsable único
- Fecha compromiso
- Criterio de cierre verificable
- Dependencias (si aplica)

## SLA de respuesta
- Mención normal: respuesta < 24h
- Mención urgente: respuesta < 4h
- Sin respuesta: escalamiento automático

## Escalamiento
1. Aviso T-2 días
2. Aviso T-1 día
3. Aviso T0 (vence hoy)
4. Vencida + escalamiento a gerencia

## KPIs semanales (máx 5)
- % tareas cerradas en fecha
- # tareas vencidas activas
- # tareas bloqueadas >48h
- Throughput semanal (# cerradas)
- Lead time promedio

## Gobernanza
- Webapp = visualización (read-only)
- Mutaciones oficiales = Gerencia IA vía chat + workflow + auditoría
